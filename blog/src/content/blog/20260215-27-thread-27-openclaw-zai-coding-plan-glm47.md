openclaw를 가볍게 써보던 때나 개인 프로젝트를 진행할 때나, 결국 먼저 보게 된 건 모델 이름보다 토큰 대비 비용과 연결성이었다. 급한 작업에서는 한 번의 출력 품질도 중요하지만, 여러 도구에서 같은 모델을 일관되게 쓸 수 있는지가 실제 체감 효율을 크게 갈랐다.

내가 glm4.7을 좋게 본 이유도 그 지점이다. 정액제인데 API key가 같이 제공돼서 붙일 수 있는 범위가 넓었고, 필요할 때는 Claude Code 백엔드로도 쓰고 오픈코드·오픈클로 환경에도 연결해 활용할 수 있었다. 한글 작업도 실사용 기준에서는 충분히 안정적이었다.

성능은 벤치마크 수치로 단정하기는 어렵다. 다만 체감상 소넷급에 가까운 구간이 있었고, 로컬 LLM과 비교하면 품질과 속도를 동시에 맞추는 데서는 우위가 분명했다. 대신 속도는 시간대에 따라 편차가 느껴져서, 촉박한 일정에서는 이 변동을 감안해 써야 했다.

가격이 첫 달 30달러 혹은 연 360달러로 제시된 점까지 포함하면, 최근 기준으로는 가성비가 좋다는 판단이 자연스러웠다. 최고 성능만 좇기보다, 여러 코딩 도구에 실제로 붙여 운영하는 관점에서 보면 꽤 실용적인 선택지였다.
